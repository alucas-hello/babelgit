BBL-022: conversational git history

---

## Why

Every work item should carry a permanent record of the conversation that produced it — decisions made, corrections given, notes left by the developer, what went wrong. This should accumulate across the entire lifecycle, not just at ship time. Even a ticket that only had one comment before closing should preserve that comment.

The goal is: future developers (and future agents) opening a work item get the full story of how and why it happened, directly in git alongside the code.

## Design decisions

**Append-only log per work item.** `.babel/conversations/BBL-XXX.md` accumulates entries over the lifetime of the WI. Each lifecycle event appends a timestamped block.

**Every lifecycle event writes an entry:** `babel todo`, `babel save`, `babel keep`, `babel refine`, `babel reject`, `babel ship`, `babel pause`, `babel continue`. Even a single comment on a todo item gets captured.

**Agent-written summary, not raw transcript.** The extraction is structured signal — decisions, corrections, files changed, what went wrong — not a verbatim dump. This is the natural sanitization layer and makes the output useful for both humans and RAG pipelines.

**Secret scrubbing before write.** Strip patterns matching env var assignments, Bearer/API tokens, connection strings, private keys, anything that looks like a secret before the file is written. The file goes into git — it must be safe.

**Pipeline is a separate concern.** babelgit's job is to put the file in git in a consistent format. How it gets into a vector DB, BullMQ queue, or any other pipeline is not babelgit's problem. The user has a private `/emb` skill that can read from git and enqueue to their own infrastructure.

## Format

```
# BBL-XXX — description

## 2026-03-23 14:02 — todo
Created by: user@email
> original description or note from conversation

## 2026-03-23 14:15 — save
Notes: what was saved
Files changed: list of modified files
Decisions: any architectural decisions made

## 2026-03-23 14:31 — ship
Verdict: ship — "user's notes"
```

## Implementation notes

- File lives at `.babel/conversations/{id}.md` — committed to the feature branch as part of each save
- Scrubber strips: `KEY=value` patterns, `Bearer \S+`, `sk-\S+`, `-----BEGIN`, connection string patterns
- Extraction is written by the agent as a structured summary of the current session's signal
- On `babel ship`, the final entry is committed and the full log is part of the branch history forever
